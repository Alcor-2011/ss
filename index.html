Cursor 与 Claude Code 入门

Cursor 是带 AI 的编辑器；Claude Code（常简写 CC）是 Anthropic 的终端里编程助手，能在项目目录里读代码、改文件、跑命令。两者可配合：在 Cursor 里写代码，在集成终端里跑 claude。

从 Cursor 开始 看 Claude Code

编辑器内 AI

聊天、Composer、Tab 补全与 @ 引用，上下文紧贴当前项目。

终端会话

在仓库根目录启动，可执行命令、改多文件，适合自动化与深度任务。

最佳实践：同一项目目录下，左边 Cursor 看 diff，右边终端跑 claude，权限与变更一目了然。

典型工作流

三步串联
1

打开项目

在 Cursor 中打开仓库根目录，保证终端当前路径与项目一致，便于 CC 读写同一套文件。

2

编辑与补全

用 Chat / Composer 讨论方案、多文件修改；用 Tab 补全快速写完样板代码。

3

终端里下任务

集成终端运行 claude，处理需要跑命令、批量改文件或管道输入的任务。

Cursor 基础功能

核心能力

AI 聊天（Chat）

侧边栏对话，理解代码与文档。适合讨论方案、解释代码。

⌘ + L（Mac）/ Ctrl + L（Windows）

Composer

多文件编辑浮层，适合一次性改多个文件的功能或重构。

⌘ + I / Ctrl + I

Cursor Tab

灰色幽灵补全，Tab 接受，适合快速写完重复结构。

@ 引用

用 @ 挂接文件、文件夹或文档，缩小上下文、提高回答质量。

规则与说明

项目内规则（如 .cursor/rules）让 AI 统一遵守风格与约束。

Claude Code（CC）怎么用

终端侧

Claude Code 在终端里工作：先安装并登录，再在项目根目录启动会话。官方文档：Claude Code 概览。

概览 CLI 用法 MCP

安装（示例）

以官方当前说明为准，常见方式包括脚本或包管理器，例如：

Shell
curl -fsSL https://claude.ai/install.sh | bash

macOS 也可用 Homebrew：

Homebrew
brew install --cask claude-code

登录与状态

claude auth login
claude auth status
  • 需有效的 Anthropic / Claude 订阅或计费方案（以账户与地区政策为准）。
  • 退出：claude auth logout

安全提示：CC 可能请求读写文件与执行命令，请仅在信任的仓库中操作，并仔细审阅每次授权。

日常使用

在仓库根目录打开终端，运行：

cd /path/to/your-project
claude

也可带首条指令启动，或用 -p 做一次性问答后退出（适合脚本）。示例：

claude "给这个仓库写一份 README 大纲"
claude -p "总结 src 目录职责"
  • claude -c：继续上一次对话（具体行为以当前 CLI 版本为准）。
  • 管道：可把文件内容 pipe 给 claude -p 做批处理分析。

与 Cursor 配合

在 Cursor 中打开项目 → 打开集成终端（Ctrl + `）→ 在同一目录运行 claude。你在编辑器里看 diff、在终端里下任务，各取所长。

  • 权限：Claude Code 会请求读写文件、执行命令等，按提示审慎批准。
  • 进阶：MCP、子代理、插件等见官方 CLI 文档。
